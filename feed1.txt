				<content:encoded><![CDATA[<p>Для сдачи задания рекомендуется использовать систему контроля версий и присылать ссылку на проект. А не просто набор файлов в почтовом сообщении.</p>
<h2>Открытые площадки</h2>
<p>Почти на всех проектах работаем с <strong>Bitbucket Server</strong> от Attlassian или с <strong>Gitlab</strong>, у заказчиков в качестве системы контроля версий используются эти продукты, чаще всего. Эти продукты можно использовать и без установки, имеются уже развёрнутые решения:</p>
<ul>
<li><a href="https://bitbucket.org/" target="_blank">https://bitbucket.org/</a></li>
<li><a href="https://gitlab.com/" target="_blank">https://gitlab.com/</a></li>
</ul>
<p>Где можно зарегистрироваться и создавать свои репозитории, команды. Репозитории могут быть открытыми и закрытыми.</p>
<p>BitBucket &#8212; часть enterprise-решения, где есть jira, confluence и всё это работает в связке. А GitLab &#8212; открытое решение, для сравнительно небольших компаний и команд. В Performance Lab нагрузочниками используется GitLab для внутренних проектов.</p>
<p>Также можно использовать очень популярное решение для открытых проектов:</p>
<ul>
<li><a href="https://github.com/" target="_blank">https://github.com/</a></li>
</ul>
<p>Если заказчик не предоставляет доступ к своему репозиторию, но нужно организовать работу команды с исходным кодом, то удобно использовать <strong>Bonobo Git Server</strong>, который просто установить и настроить на одной из нагрузочных станций, чаще всего нагрузочные станции работают под <strong>Microsoft Windows Server</strong>.</p>

<ul>
<li><a href="https://bitbucket.org/owasp" rel="nofollow">https://bitbucket.org/owasp</a></li>
</ul>
<p>На GitLab.com пользователь, которого нужно добавить в команду проекта:</p>
<ul>
<li><a href="https://gitlab.com/smirnov_va" rel="nofollow">https://gitlab.com/smirnov_va</a></li>
</ul>
<p>На GitHub.com закрытый проект не создать просто так, поэтому там никого в команду добавлять не надо, и так будет всё видно. Мой пользователь там:</p>
<ul>
<li><a href="https://github.com/polarnik" rel="nofollow">https://github.com/polarnik</a></li>

<h2>Работа с репозиторием</h2>
<p>При создании репозитория на сайте bitbucket/gitlab/github процесс может отличаться.</p>
<p>А при работе с репозиторием процесс всегда одинаковый:</p>
<ul>
<li>можно использовать консольный клиент;</li>
<li>можно использовать графический клиент.</li>
</ul>
<p>Наиболее удобным графическим клиентом для Windows считаю клиент от Atlassian:</p>
<ul>
<li><a href="https://www.sourcetreeapp.com/" target="_blank">https://www.sourcetreeapp.com/</a></li>
</ul>
<p>А также стандартный клиент git-gui и стандартное приложение gitk, они успешно работают даже если нет подключения к сети Интернет.</p>
<p>Широкий список клиентов доступен на странице:</p>
<ul>
<li><a href="https://git-scm.com/downloads/guis" target="_blank">https://git-scm.com/downloads/guis</a></li>
</ul>
<h2>SourceTree + WinMerge</h2>
<p>Хорошая связка для сохранения результатов разработки <strong>JMeter</strong> в <strong>git</strong>:</p>
<ul>
<li><strong>SourceTree</strong> для работы с Git-репозиторием (<a href="https://www.sourcetreeapp.com/">https://www.sourcetreeapp.com/</a>);</li>
<li><strong>WinMerge</strong> для сравнения jmx-документов, вызывается по &lt;<em>Ctrl+D&gt;</em> из <strong>SourceTree</strong> (<a href="http://winmerge.org/">http://winmerge.org/</a>).</li>
</ul>
<p><strong>WinMerge</strong> позволяет задать такие настройки, при которых отображение отличий в XML-документах выглядит более простым и удобным, чем в стандартной утилите <strong>diff</strong>.</p>

<h2>Ссылка на версию</h2>
<p>Присылать на проверку можно как ссылку на репозиторий. Так и ссылку на конкретную версию.</p>
<p>С помощью команды <a href="https://git-scm.com/docs/git-tag" target="_blank">git tag</a> или возможностей графического клиента задаётся метка, например, &#171;v0.01&#187;. И далее удобно отправлять на проверку проект указанной версии и продолжать работу над проектом.</p>
<p>Примеры таких ссылок для bitbucket.org:</p>
<ul>
<li><a href="https://bitbucket.org/atlassian-marketplace/jmeter-sbt-plugin/downloads/?tab=tags" target="_blank">https://bitbucket.org/atlassian-marketplace/jmeter-sbt-plugin/downloads/?tab=tags</a>.</li>
</ul>
<p>Примеры таких ссылок для gitlab.com:</p>
<ul>
<li><a href="https://gitlab.com/gitlab-org/gitlab-ce/tags" target="_blank">https://gitlab.com/gitlab-org/gitlab-ce/tags</a>.</li>
</ul>
<p>Примеры таких ссылок для github.com:</p>
<ul>
<li><a href="https://github.com/pflb/Jmeter.Plugin.TailSampler/releases" target="_blank">https://github.com/pflb/Jmeter.Plugin.TailSampler/releases</a>;</li>
<li><a href="https://github.com/pflb/Jmeter.Plugin.TailSampler/tags" target="_blank">https://github.com/pflb/Jmeter.Plugin.TailSampler/tags</a>.</li>

				<content:encoded><![CDATA[<p>Текущий сайт https://loadtestweb.wordpress.com/ доступен по протоколу HTTPS. И задания предполагают работу с HTTP-запросами, которые удобно изучать используя Fiddler 4.</p>
<p>Ниже описано, как его настроить и использовать на примере следующего окружения:</p>
<ul>
<li>Windows 7 x64 (операционная система);</li>
<li>.NET Framework 4 (предполагается, что .NET 4 установлен в системе, нужен для работы Fiddler 4);</li>
<li>Fiddler 4 (прокси сервер);</li>
<li>Mozilla Firefox (браузер, который будет работать через прокси-сервер).</li>
</ul>
<p>Дистрибутив Fiddler 4 можно скачать по ссылке:</p>
<ul>
<li><a href="https://www.telerik.com/docs/default-source/fiddler/fiddlersetup.exe?sfvrsn=4" target="_blank">https://www.telerik.com/docs/default-source/fiddler/fiddlersetup.exe?sfvrsn=4</a></li>
</ul>
<p>Описание доступно на сайте проекта:</p>
<ul>
<li><a href="http://www.telerik.com/fiddler" target="_blank">http://www.telerik.com/fiddler</a></li>
</ul>
<p>После установки можно сделать следующие натройки:</p>
<ul>
<li>сделать прокси-сервис несистемным;</li>
<li>включить расшифровку HTTPS;</li>
<li>отключить проверку обновлений.</li>
</ul>
<h1>Настройки Fiddler 4</h1>
<h2>Сделать прокси-сервис несистемным</h2>
<p>Запустить <strong>Fiddler 4</strong> и отжать кнопку перехвата всех HTTP-запросов, чтобы не было надписи <em><span style="text-decoration:underline;"><strong>Capturing</strong></span></em>.</p>

<ul>

<li><em><span style="text-decoration:underline;"><strong>Decrypt HTTPS traffic</strong></span></em>.</li>
</ul>

				<content:encoded><![CDATA[<h1>Интенсивность и профиль нагрузки</h1>
<p>Чтобы после отправки комментария не возвращалось сообщение:</p>
<blockquote><p>Вы комментируете слишком быстро. Попридержите коней.</p></blockquote>
<p>Надо чтобы сценарии выполнялись с суммарной интенсивностью 1,5-2,5 сценария в минуту. То есть все виртуальные пользователи, должны создавать нагрузку соотвествующую 1,5-2,5 сценария в минуту.</p>
<p>В сценарии должно отправляться два комментария:</p>
<ol>
<li>Комментарий от незарегистрированного пользователя:
<ul>
<li>Использовать список из 10-ти пользователей вида:
<ul>
<li><a href="mailto:tectep1@loadtestweb.wordpress.com">tectep1@loadtestweb.wordpress.com</a>, Performance Tester 01, https://loadtestweb.wordpress.com/</li>
<li><a href="mailto:tectep2@loadtestweb.wordpress.com">tectep2@loadtestweb.wordpress.com</a>, Performance Tester 02, <a href="https://loadtestweb.wordpress.com/" rel="nofollow">https://loadtestweb.wordpress.com/</a></li>
<li><a href="mailto:tectep3@loadtestweb.wordpress.com">tectep3@loadtestweb.wordpress.com</a>, Performance Tester 03, <a href="https://loadtestweb.wordpress.com/" rel="nofollow">https://loadtestweb.wordpress.com/</a></li>
<li>&#8230;</li>
</ul>
</li>
<li>Параметры пользователей загружать, используя <strong>CSV Data Set Config</strong>, описание способа смотри ниже.</li>
</ul>
</li>
<li>Комментарий от зарегистрированного пользователя, одного:
<ul>
<li>Использовать только одного пользователя:
<ul>
<li>Выполнить аутентификацию через wordpress.com (учётные данные будут созданы)</li>
<li>Через twitter (необходимо зарегистрироваться самостоятельно)</li>
<li>Через Google (необходимо зарегистрироваться самостоятельно)</li>
<li>Через Facebook (необходимо зарегистрироваться самостоятельно)</li>
</ul>
</li>
<li>Параметры пользователя загружать, используя также <strong>CSV Data Set Config</strong> с файлом из одной строки или переменные и файл свойств, описание способов смотри ниже.</li>
</ul>
</li>
</ol>
<p>Между первым и вторым комментарием надо поставить паузу, задержку, миниумм на 10-15 секунд. Паузы в <strong>Apache.Jmeter</strong> создаются с помощью таймеров.</p>
<p>Чтобы проверить, что интенсивность расчитана верно. Можно ориентироваться на то, что каждую минуту должно появляться под тестовой страницей примерно 4 комментария (+/-).</p>
<p>У каждого учащегося своя <a href="https://loadtestweb.wordpress.com/category/%D1%82%D0%B5%D1%81%D1%82%D0%BE%D0%B2%D0%B0%D1%8F-%D1%81%D1%82%D1%80%D0%B0%D0%BD%D0%B8%D1%86%D0%B0/" target="_blank">тестовая страница</a> и своё количество виртуальных пользователей.</p>
<h1>Cookies</h1>
<p>Важно перед каждой итерацией очищать cookies. Иначе единожды вошедший пользователь, например, через wordpress, так всегда и будет считаться вошедшим.</p>
<p>А нужно каждую итерацию выполнять одинаковый сценарий. То есть, выполнять аутентификацию снова и снова.</p>
<p>Параметры очистки Cookies задаются в конфигурационном элементе <strong>HTTP Cookie Manager</strong>:</p>
<ul>
<li><a href="http://jmeter.apache.org/usermanual/component_reference.html#HTTP_Cookie_Manager" target="_blank">HTTP Cookie Manager</a>.</li>
</ul>
<h1>Тексты комментариев</h1>
<p>Тексты комментариев можно подобрать самостоятельно, можно взять готовые из списка сформированного на основе пословиц и поговорок из русскоязычной википедии:</p>
<ul>
<li><a title="comments-1" href="https://loadtestweb.files.wordpress.com/2017/03/comments-1.xls">comments-1</a> (документ <strong>Excel</strong>, на данном сайте не разметить txt-файл, в <strong>JMeter</strong> нужно использовать txt-файл или csv-файл из одной колонки).</li>
</ul>
<p>В файле тексты отсортированы в алфавитном порядке. Чтобы имитировать процесс отправки случайных комментариев из некоторого набора можно перемешать строки в файле случайным образом перед запуском тестирования. Например, используя текстовый редактор <strong>Sublime Text 2</strong> или в <strong>Microsoft Office Excel</strong> (добавить колонку со случайным значением и отсортировать по ней).</p>

<p>Для загрузки текстов из файла в переменную рекомендуется использовать <strong>HTTP Simple Table Server</strong>:</p>
<ul>
<li><a href="https://jmeter-plugins.org/wiki/HttpSimpleTableServer/" target="_blank">HTTP Simple Table Server</a>.</li>
</ul>
<p>Причина почему не подойдёт стандартный компонент <strong>CSV Data Set Config</strong> &#8212; сервер проверяет, то, что пользователь уже отправлял комментарий с таким же текстом. А если дважды запустить тест, который берёт комментарии из файла с помощью <strong>CSV Data Set Config</strong>, то дважды начнут выбираться комментарии из начала файла. И при повторной отправке wordpress вернёт ответ &#8212; такой комментарий уже был и комментарий не добавится.</p>
<blockquote><p>Ошибка при отправке комментария</p>
<p>Обнаружен дубликат комментария. Кажется, вы уже сказали это!</p></blockquote>
<p>А с использованием <strong>HTTP Simple Table Server</strong> можно организовать ротацию комментариев. Читать комментарии из начала и добавлять их в конец таблицы. Комментариев 3675 в файле. За 5-ти минутный тест отправляется 10-15 комментариев. Поэтому файла хватит на сотни отладочных запусков.</p>
<p>Вторая причина использования <strong>HTTP Simple Table Server </strong>в том, что в <strong>HP LoadRunner</strong> работа с тестовыми данными осуществляется через <strong>Virtual Table Server</strong>. И чтобы привыкнуть к такому подходу, и сравнить удобство работы <strong>Jmeter</strong> vs <strong>LoadRunner</strong> тут рекомендуется использовать <strong>HTTP Simple Table Server</strong>.</p>
<h1>Логин, email, страница &#8230;</h1>
<h2>Вариант через <strong>CSV Data Set Config</strong></h2>
<p>Можно для загрузки всех данных использовать компонент <strong>CSV Data Set Config</strong>:</p>
<ul>
<li>Один для выбора значений полей для комментирования (в файле будут строки с тремя колонками тестовых данных, в данном случае):
<ul>
<li>email</li>
<li>author</li>
<li>url</li>
</ul>
</li>
<li>Один для добавления авторизованного комментария (в файле будет одна пара логин/пароль):
<ul>
<li>login</li>
<li>password</li>
</ul>
</li>
<li>Также можно создать файл со списком страниц, к которым нужно оставить комментарий и также загружать это значение из файла:
<ul>
<li>адрес страницы, к которой надо оставлять комментарий (<code>/2017/02/25/test-9/</code>)</li>
</ul>
</li>
</ul>
<p>Преимущества: теоретически можно расширять состав учётных данных, которые будут использоваться для комментирования и тест будет отправлять комментарии уже от двух, трёх, &#8230; пользователей к двум, трём, &#8230; страницам.</p>
<h2>Вариант через переменные и файл свойств</h2>
<p>Если не заглядывать в будущее и не закладывать возможность расширения списка учётных данных и страниц, то можно все значения задать в единственном экземпляре. Используя для этого переменные.</p>
<p>Можно добавить значения параметров в конфигурационный файл. Например, назвать его <strong>loadtest.properties</strong>, для примера смотри файл <code>apache-jmeter-3.1\bin\user.properties</code>, заполнить файл как-то так:</p>
<pre><code>loadtest.testPage=/2017/02/25/test-9/

loadtest.email=tectep@inbpx.ru
loadtest.author=tectep
loadtest.url=http://mail.ru

loadtest.worpress.login=secretLogin
loadtest.worpress.password=************
</code></pre>
<p>Разместить файл рядом с файлом скрипта. А в bat-файл запуска <strong>JMeter</strong> добавить параметр: <code>--addprop loadtest.properties</code></p>
<p>После чего все properties, указанные в файле, будут доступны в <strong>JMeter</strong>, с помощью функции получения значений свойств <strong>property</strong>:</p>
<pre><code>${__property(loadtest.testPage,,/2017/02/25/test-9/)}
${__property(loadtest.email,,tectep@inbpx.ru)}
${__property(loadtest.author,,tectep)}
${__property(loadtest.url,,http://mail.ru)}
${__property(loadtest.worpress.login,,login)}
${__property(loadtest.worpress.password,,*****)}
</code></pre>
<p>Используя таблицу <strong>User Defined Variables</strong> в корневом компоненте <strong>Test Plan</strong> можно сразу задавать значения нужных переменных. В качестве значения брать значение из файла с properties.</p>
<p>Например задать переменную с именем:</p>
<pre><code>loadtest_testPage</code></pre>
<p>В качестве значения указать:</p>
<pre><code>${__property(loadtest.testPage,,/2017/02/25/test-9/)}</code></pre>
<p>А потом получить значения переменной в тесте, везде, где нужно:</p>
<pre><code>${loadtest_testPage}</code></pre>
<h1>Корреляция параметров формы</h1>
<p>При отправке комментария серверу передаётся большое количество полей. Среди них есть несколько CSRF-токенов, их имена содержат строку &#171;nonce&#187;. CSRF-токены генерируются индивидуально для пользователя, открывшего страницу и ожидается, что пользователь передаст их серверу вместе с комментарием. Это будет признаком того, что пользователь действительно открыл страицу сайта и не является роботом-спаммером, отправляющим комментарии.</p>
<p>Эти токены нужно обязательно скоррелировать:</p>
<ol>
<li>Найти в ответе на какой запрос они приходят от сервера. Удобно воспользоваться консолью браузера или использовать полнотекстовый поиск в прокси-сервере Fiddler4:
<ul>
<li><a href="https://www.telerik.com/docs/default-source/fiddler/fiddlersetup.exe?sfvrsn=4" target="_blank">Fiddler4</a>.</li>
</ul>
</li>
<li>Добавить в нагрузочный скрипт компонент для выполнения такого запроса:
<ul>
<li><a href="http://jmeter.apache.org/usermanual/component_reference.html#HTTP_Request" target="_blank">HTTP Request</a>.</li>
</ul>
</li>
<li>Добавить Post Processor-ы, для извлечения значений полей, которые нужно передавать при отправке комментариев. Рекомендуется использовать:
<ul>
<li><a href="http://jmeter.apache.org/usermanual/component_reference.html#CSS/JQuery_Extractor" target="_blank">CSS/JQuery Extractor</a> (простой и понятный синтаксис селекторов);</li>
<li><a href="http://jmeter.apache.org/usermanual/component_reference.html#Regular_Expression_Extractor" target="_blank">Regular Expression Extractor</a> (синтаксис сложнее, но очень быстро работает).</li>
</ul>
</li>
<li>Добавить в нагрузочный скрипт компонент для отправки комментария, с передачей CSRF-токенов, идентификатора статьи к которой отправляется комментарий, текста комментария, информации о пользователе.</li>
</ol>

<ul>

</ul>
</li>
<li><strong>Attribute</strong>: <code>value</code></li>
<li><strong>Match No. (0 for Random)</strong>: <code>1</code></li>
<li><strong>Default Value</strong>:</li>
</ul>

				<content:encoded><![CDATA[<p>Есть <a href="https://loadtestweb.wordpress.com/category/%d1%82%d0%b5%d1%81%d1%82%d0%be%d0%b2%d0%b0%d1%8f-%d1%81%d1%82%d1%80%d0%b0%d0%bd%d0%b8%d1%86%d0%b0/" target="_blank">набор тестовых страниц</a>. У каждого своя страница для экспериментов.</p>
<p>Задача: имитировать в Apache.JMeter открытие своей статьи.</p>
<p>Например, при открытии статьи <a href="https://loadtestweb.wordpress.com/2017/03/01/%d0%b7%d0%b0%d0%b4%d0%b0%d0%bd%d0%b8%d0%b5-1-%d1%80%d0%b0%d0%b1%d0%be%d1%82%d0%b0-%d1%81-xml-%d0%b2-jmeter/" target="_blank">Задание 1. Работа с XML в JMeter</a> браузер <strong>Mozilla Firefox</strong> отправляет 102 запроса.</p>
<p><a href="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png"><img data-attachment-id="224" data-permalink="https://loadtestweb.wordpress.com/2017/03/02/%d0%b7%d0%b0%d0%b4%d0%b0%d0%bd%d0%b8%d0%b5-3-%d0%b7%d0%b0%d0%b3%d1%80%d1%83%d0%b7%d0%ba%d0%b0-html-%d1%81%d1%82%d1%80%d0%b0%d0%bd%d0%b8%d1%86%d1%8b-%d0%b2-apache-jmeter/2017-03-02_01-04-47/" data-orig-file="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=1140" data-orig-size="1360,1012" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="2017-03-02_01-04-47" data-image-description="" data-medium-file="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=1140?w=300" data-large-file="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=1140?w=1024" class="alignnone size-full wp-image-224" src="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=1140" alt="2017-03-02_01-04-47" srcset="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=1140 1140w, https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=150 150w, https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=300 300w, https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=768 768w, https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png?w=1024 1024w, https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-04-47.png 1360w" sizes="(max-width: 1140px) 100vw, 1140px"   /></a></p>
<p>Один запрос для загрузки html-содержимого страницы. И 101 дополнительный запрос.</p>
<p>Не каждый дополнительный запрос важен для нагрузочного тестирования. Можно проанализировать запросы с каким доменам выполняются. И если осуществляется нагрузочное тестирование конкретного сервиса, то запросы к внешним системам, часто нужно исключить из профиля нагрузки. В нашем случае, есть запросы к facebook, к wordpress, к gavatar, &#8230;</p>
<p><a href="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png"><img data-attachment-id="226" data-permalink="https://loadtestweb.wordpress.com/2017/03/02/%d0%b7%d0%b0%d0%b4%d0%b0%d0%bd%d0%b8%d0%b5-3-%d0%b7%d0%b0%d0%b3%d1%80%d1%83%d0%b7%d0%ba%d0%b0-html-%d1%81%d1%82%d1%80%d0%b0%d0%bd%d0%b8%d1%86%d1%8b-%d0%b2-apache-jmeter/2017-03-02_01-15-24-2/" data-orig-file="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png?w=1140" data-orig-size="766,479" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="2017-03-02_01-15-24" data-image-description="" data-medium-file="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png?w=1140?w=300" data-large-file="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png?w=1140?w=766" class="alignnone size-full wp-image-226" src="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png?w=1140" alt="2017-03-02_01-15-24" srcset="https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png 766w, https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png?w=150 150w, https://loadtestweb.files.wordpress.com/2017/03/2017-03-02_01-15-241.png?w=300 300w" sizes="(max-width: 766px) 100vw, 766px"   /></a></p>
<p>Запросы к каждой из группы доменов вносит вклад в длительность открытия страницы. Но часто к некоторым доменам отправлять запросы во время нагрузочного тестирования ну нужно. Особенно при большой интенсивности запросов. Такое решение может быть принято на усмотрение инженера, разрабатывающего тест. Или может быть требование со стороны заказчика или владельца какой-то группы доменов.</p>
<p>Чтобы эмулировать загрузку встроенных ресурсов в компоненте <strong>HTTP Request</strong> есть возможность распарсить HTML/CSS-содержимое и загрузить объекты по ссылкам:</p>
<ul>
<li><a href="http://jmeter.apache.org/usermanual/component_reference#HTTP_Request" target="_blank">HTTP Request</a></li>
</ul>

<p>Задача: сэмулировать в <strong>Apache.JMeter</strong> нажатие кнопки <em><span style="text-decoration:underline;"><strong>Предыдущие записи</strong></span></em>, обработать ответ (получить список предыдущих записей), и открыть случайную запись из списка предыдущих.</p>
<p>Это будет не тоже самое что делает браузер &#8212; браузер подгружает заголовки и миниатюры всех страниц из json-документа, делает несколько последующих запросов. А тестовое задание предполагает загрузку одной полной статьи из списка предыдущих, а не загрузку нескольких миниатюр. Тестовое задание обыгрывает более простую ситуацию.</p>
<h1>Как записать запрос</h1>
<p>Чтобы записать POST-запрос, выполняемый при нажатии кнопки <em><span style="text-decoration:underline;"><strong>Предыдущие записи</strong></span></em>, рекомендую использовать <strong>Fiddler 4</strong>. Как настроить работу браузера, через прокси-сервер написано в инструкции: <a href="https://loadtestweb.wordpress.com/2017/03/02/fiddler-4-%d0%bd%d0%b0%d1%81%d1%82%d1%80%d0%be%d0%b9%d0%ba%d0%b0-%d0%b8-%d0%b8%d1%81%d0%bf%d0%be%d0%bb%d1%8c%d0%b7%d0%be%d0%b2%d0%b0%d0%bd%d0%b8%d0%b5-%d1%81-https/" target="_blank">Fiddler 4. Настройка HTTPS</a>.</p>
<p>При формировании запроса важны:</p>
<ul>
<li>адрес</li>
<li>тип запроса &#8212; POST</li>
<li>набор и значения параметров</li>
</ul>

<h1>Выбор случайной ссылки из JSON-ответа</h1>
<h2>Применение JSON Path-выражений к JSON-документу</h2>
<p>С помощью <strong>JSON </strong><strong>Extractor</strong> (<em>Post </em><em>Processor/ </em><em>JSON </em><em>Extractor</em>) нужно продемонстрировать извлечение ссылки на случайную статью.</p>
<p>Для отладки json-выражения можно использовать инструменты:</p>
<ul>
<li><a href="http://jsonprettyprint.com" target="_blank">http://jsonprettyprint.com</a> для форматирования json-ответа, так проще написать селектор к json;</li>
<li><a href="http://www.jsonquerytool.com/">http://www.jsonquerytool.com/</a> для отладки селектора, синтаксис селекторов есть в документации по ссылкам ниже.</li>
</ul>

<h2>Применение регулярных выражений</h2>
<p>В данном случае, возможностей JSON Path-выражений недостаточно, для получения нужной ссылки (мне так кажется, если сумеете сделать выборку ссылки только на JSON Path &#8212; будет здорово).</p>
<p>Значение ссылки является именем поля, заключенным в двойные кавычки. Для извлечения текста между двумя двойными кавычками можно использовать регулярные выражения:</p>
<ul>
<li><a href="http://jmeter.apache.org/usermanual/regular_expressions.html" target="_blank">http://jmeter.apache.org/usermanual/regular_expressions.html</a></li>
</ul>
<p>А возможность компонента <strong>Regular Expression Extractor</strong> позволяет сразу выбрать случайное значение из результатов выборки:</p>
<ul>
<li><strong><a href="http://jmeter.apache.org/usermanual/component_reference.html#Regular_Expression_Extractor" target="_blank">Regular Expression Extractor</a></strong>.</li>
</ul>
<p>Регулярное выражение можно применить к значению переменной. <strong>Regular Expression Extractor</strong> (<em>Post Processor/ Regular Expression Extractor</em>) может работать со значениями указанной переменной (поле <strong>Jmeter Variable</strong>).</p>
<p>Значит можно в <strong>JSON Path Extractor</strong> извлечь нужный кусок JSON-документа в переменную, а потом извлечь нужный случайный кусок из переменной с помощью <strong>Regular Expression Extractor</strong>.</p>
<h2>Примение javaScript на примере обратного экранирования</h2>
<p>В инструменте <a href="http://www.jsonquerytool.com" rel="nofollow">http://www.jsonquerytool.com</a> автоматически выполнится обратное экранирование и ссылки будут выглядеть сразу так:</p>
<pre>"https://loadtestweb.wordpress.com/2017/02/25/test-5/"</pre>
<p>В компоненте <strong>JSON Path Extractor</strong> автоматического обратного экранирования не происходит и ссылки выглядят так:</p>
<pre>"https:\/\/loadtestweb.wordpress.com\/2017\/02\/25\/test-5\/"</pre>
<p>Пусть с помощью регулярного выражения мы извлекли случайную ссылку в переменную <strong>postflairURL</strong>.</p>
<p>Тогда с помощью функции <strong>__javaScript</strong> можно избавиться от экранирования, получив значение строки:</p>
<pre>${__javaScript(${postflairURL},)}</pre>
<p>Документация:</p>
<ul>
<li>функция <strong><a href="http://jmeter.apache.org/usermanual/functions.html#__javaScript" target="_blank">__javaScript</a></strong>.</li>
</ul>
<p>Можно полностью выборку ссылки из json-документа сделать на javaScript и тогда не надо будет использовать <strong>JSON Path Extractor</strong> и <strong>Regular Expression Extractor</strong>. Для учебного примера важно задействовать как можно больше стандартных блоков <strong>Apache.JMeter</strong>, поэтому тут предполагается извлечение ссылки в три этапа:</p>
<ol>
<li><strong>JSON Path Extractor</strong> для извлечения части json-документа.</li>
<li><strong>Regular Expression Extractor</strong> для извлечения текста между двойными кавычками.</li>
<li><strong>javaScript</strong> для обратного экранирования слешей в тексте.</li>
</ol>

<p>Чтобы открыть старницу по ссылке, снова используется компонент <strong>HTTP Request</strong>:</p>
<ul>
<li><strong><a href="http://jmeter.apache.org/usermanual/component_reference.html#HTTP_Request" target="_blank">HTTP Request</a></strong>.</li>

И просматрировать значения переменных во <strong>View Result Tree</strong>. То есть для отладки удобно использовать метод &#171;отладочная печать&#187; и связку стандартных компонент:</p>
<ul>
<li><a href="http://jmeter.apache.org/usermanual/component_reference.html#Debug_PostProcessor" target="_blank">Debug PostProcessor</a>;</li>

<li>Запустить менеджер плагинов <em>&lt;Главное меню / Options / Plugin Manager&gt;</em>.</li>
<li>Установить <a href="https://github.com/Blazemeter/jmeter-debugger" target="_blank">BlazeMeter Step-by-step Debugger</a> отметив компонент и нажав кнопку <em><span style="text-decoration:underline;"><strong>Apply Changes And Restart Jmeter</strong></span></em>.</li>
<li>Теперь можно отлаживать скрипт в отладчике и просматрировать значения переменных, формирумых в результате работы Post Processor-ов, таких как <strong>XPath Extractor</strong>, в отладчике.</li>

<h2>Профиль нагрузки</h2>
<p>Сценарий работы:</p>
<ol>
<li>Открыть стартовую страницу сайта.</li>
<li>Загрузить RSS-ленту, выбрать из RSS ссылку на случайную статью и открыть статью по выбранной ссылке. <em>Демонстрация работы с XML.</em></li>
<li>Нажать кнопку <em><span style="text-decoration:underline;"><strong>Предыдущие записи</strong></span></em> на стартовой странице, из JSON-ответа сервера выбрать ссылку на одну из предыдущих статей и открыть статью по выбранной ссылке. <em>Демонстрация работы с JSON и RegExp.</em></li>
<li>Открыть статью по ссылке, указанной как персональная тестовая страница, заданная для каждого ученика.</li>
<li>Оставить комментарий к тестовой странице, неавторизованным пользователем. <em>Работа с CSS-селекторами.</em></li>
<li>Пауза 10-15 секунд минимум.</li>
<li>Оставить комментарий к тестовой странице, выполнив аутентификацию. <em>Работа с CSS-селекторами.</em></li>
<li>Пауза за счёт ожидания завершения шага нагрузки, шаг нагрузки нужно рассчитать.</li>
</ol>
<p>Подобрать шаг нагрузки так, чтобы суммарная интенсивность выполнения такого сценария была 1,5-2,5 сценария в минуту.</p>
<p>Чтобы расчёт шага нагрузки не повторялся, у каждого варианта выполнения задания своё количество VU (виртуальных пользователей):</p>
